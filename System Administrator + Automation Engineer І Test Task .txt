• Мова на ваш вибір: Python, Node.js, PHP, Go або інша, з якою ви впевнено працюєте. • Зберегти результат у Google Sheets (через Google Sheets API) АБО в CSV з інструкцією, як занести в Sheets. 
• Реалізувати повторний запуск без дублювання (upsert по unique key). 
• Додати логування: час запуску, кількість зібраних/нових/оновлених записів, помилки. 
• Описати в README, як це розгорнути на Windows-сервері і запускати за розкладом (Task Scheduler або GitHub Actions). 
• Передати: репозиторій GitHub + README + скріншот успішного запуску.
Орієнтовно це робота на 1 вечір. Оцінюємо: чистоту коду, обробку помилок, розуміння upsert, якість документації.
⏱️ Дедлайн та формат виконання
• Термін виконання: Оскільки об'єм роботи орієнтовно розрахований на 1 вечір, буде супер, якщо вдасться надіслати результат протягом 3 робочих днів. Якщо тобі знадобиться більше часу через поточну зайнятість, просто попередьте, це абсолютно ок.
• Формат: Чекаємо на посилання на твій GitHub-репозиторій (із README файлом), а також скріншот успішного запуску скрипту з логами.
🚀 Що далі?
Після отримання виконаного завдання ми з командою детально його вивчимо. Перевірка зазвичай займає від 1 до 5 робочих днів, після чого ми обов'язково повернемося із фідбеком, у випадку якщо він позитивний, то ми запропонуємо зручний час для зідзвону, щоб познайомитися ближче.
